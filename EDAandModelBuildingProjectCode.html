FinalProject

It can often be difficult to determine whether or not a candidate is suitable for a loan given many different factors and variables that may differ slightly from past candidates. Given historical data from LendingClub, a US peer-to-peer lending company, on loans from their company, we can analyze this data to build a model that will predict whether or not a given loan will be repaid by the borrower. This will streamline the screening process of new loan applicants, who, given the necessary information, will be accepted or rejected for a loan.

Model Fitting

Model fitting was not an easy task with this dataset. R struggles to train a model with hundreds of thousands of records in a reasonable amount of time. Additionally, our exploratory analysis revealed that the target variable is unbalanced. Before model training, we wanted our dataset to be smaller and more balanced for the target variable.

To create the balanced dataset, we gather all the rows from the minority class (loan_status = 0) and then grab a sample of the same size as the minority class from the majority class (loan_status = 1). Then, merge the two into a single dataframe.

loan_status_zero = data[data$loan_status == 0, ]
loan_status_one_selected = sample(1:nrow(data[data$loan_status == 1, ]), size=nrow(loan_status_zero), replace=FALSE)
loan_status_one = data[data$loan_status == 1, ][loan_status_one_selected,]
balanced_data = rbind(loan_status_zero, loan_status_one)

Then, we create our training and testing sets from the balanced dataset.

train.test.partition <- function(data, train.percent){
   selected <- sample(1:nrow(data), round(nrow(data)*train.percent), replace=FALSE)
   train <- data[selected,]
   rest <- setdiff(1:nrow(data), selected)
   test <- data[rest,]
   return(list(train=train, test=test))
}
